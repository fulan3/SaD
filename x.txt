Saya sudah memperbaiki routing aplikasi React Vite saya
agar bisa deploy ke GitHub Pages.
Sekarang routing SUDAH BENAR dan JANGAN DIUBAH LAGI.

Masalah saya sekarang:
- gambar (img, svg)
- background-image di CSS
- asset statis lain (icon, font)

masih berpotensi blank / corrupt setelah deploy.

Repo GitHub:
fulan3/SAD

Tugas kamu (FOKUS ASSET SAJA):
1. Audit semua penggunaan img, svg, css, dan asset statis
2. Ubah SEMUA path absolut (contoh: "/img/logo.png")
   menjadi path yang aman untuk GitHub Pages
3. Gunakan salah satu pola yang benar:
   - import asset sebagai module (import logo from ...)
   - atau `import.meta.env.BASE_URL` untuk asset di public/
4. Pastikan asset dari folder `public/` tetap bisa diakses
5. Pastikan background-image di CSS tidak 404
6. Jangan mengubah routing, Router, atau basename
7. Jangan menambahkan library baru
8. Jangan mengubah struktur UI atau logic aplikasi

Output yang saya inginkan:
- Contoh kode SEBELUM vs SESUDAH untuk img & css
- Checklist asset-safe untuk GitHub Pages
- Penjelasan singkat kenapa asset bisa blank walau routing sudah benar
- Aplikasi tetap bisa diakses di:
  https://fulan3.github.io/SAD/
